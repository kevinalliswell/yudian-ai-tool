# 09 · 必须规避的坑

1. 坏配置必须兜底并记日志，不能启动崩溃。
2. dPt 失败必须默认 `1/1`。
3. 连接判断和使用不能分离；用 Actor 串行状态。
4. 刷新间隔下限 200ms。
5. 曲线段循环出错也要节流。
6. 越界拒绝，不能截断。
7. 补码转换先判范围再转换。
8. 禁止静默吞异常。
9. 设备 IO 不阻塞 UI，超时约 300ms。
10. 校验范围单一来源 Rust。
11. 实时数据事件推送，前端不轮询 invoke。
12. 配置/预设用 plugin-store。
13. 运行期敏感配置不入库。
14. 图标与平台资源必须齐全；CI 任一平台失败即失败。
